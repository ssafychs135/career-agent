# 전역 필터(지역·기업) 설계

**작성일:** 2026-07-23
**목표:** 공고 목록에 나오기 전에 걸리는 전역 필터를 만든다. 지역은 **허용목록**(이 지역만 표시), 기업은 **숨김목록**(전체 기업 목록에서 선택 해제한 기업을 감춤).

## 배경 / 현재 데이터

Explorer에는 세션 단위 임시 필터(지역 2단계·연차·검색)가 이미 있지만, **저장되는 전역 규칙**은 없다. 매번 같은 기업을 눈으로 걸러내야 한다.

측정한 현재 데이터(2026-07-23, 프로덕션 446건):

| 항목 | 값 |
|---|---|
| 전체 공고 | 446 |
| 지역 정보 없는 공고 | **0** |
| 기업 수 | **255** |
| 지역 | 서울 362 · 경기 59 · 인천 5 · 대전 5 · 대구 4 · 전북 4 · 부산 4 · 울산 2 · 세종 1 (9개) |

이 수치가 UI를 규정한다: **기업 255개는 단순 나열로 못 쓴다**(검색·정렬 필수), **지역 9개는 체크박스 한 줄로 충분**하다.

## 결정 사항 (확정)

1. **적용 시점: 조회 시점 숨김.** 수집·요약은 그대로 하고 목록 쿼리에서만 제외한다. 필터 변경이 즉시 반영되고 언제든 되돌릴 수 있다.
2. **방향: 지역=포함(허용목록), 기업=제외(숨김목록).**
3. **기업 선택 방식: 전체 목록에서 체크 해제.** 텍스트 입력이 아니라 실제 데이터에서 고르므로 **정확일치**가 정답이 되고 매칭 모호성이 사라진다.
4. **지역·기업을 한 페이지(`/filters`)에** 모은다.
5. **공고 상세는 손대지 않는다.** 필터는 목록 쿼리에만 적용된다. 열어둔 공고의 기업을 숨겨도 목록에서만 사라지고 상세는 그대로 — 숨김은 삭제가 아니므로 의도된 동작이며, 별도 코드가 필요 없다.

## 데이터 모델

`app_settings`(싱글턴)에 두 컬럼 추가 — 마이그레이션 `0005_global_filter`.

| 컬럼 | 타입 | 의미 | 빈 배열일 때 |
|---|---|---|---|
| `allowed_regions` | `text[] NOT NULL DEFAULT '{}'` | 이 지역만 표시 | **전체 표시**(필터 미적용) |
| `hidden_companies` | `text[] NOT NULL DEFAULT '{}'` | 이 기업 숨김 | 아무것도 숨기지 않음 |

`Settings` 모델(pydantic)과 `_COLUMNS` UPSERT 목록에 동일하게 추가한다.

## 필터 적용 — `build_list_query`

목록 쿼리에만 두 절을 추가한다(전부 `$N` 파라미터, 값 보간 없음).

**지역(허용목록)** — 공고의 `locations`("서울 강남구, 경기 성남시" 형태의 단일 텍스트) 중 하나라도 허용 지역이면 통과:

```sql
(locations ILIKE $1 OR locations ILIKE $2)   -- $n = '%서울%', '%경기%'
```

정규화된 지역 테이블이 없으므로 기존 `location` 필터와 동일하게 ILIKE로 맞춘다. 지역명(서울/경기/부산…)이 서로 구별되어 오탐 위험이 실질적으로 없다.

**기업(숨김목록)** — 정확일치 제외. `company`가 NULL이면 `NOT (company = ANY(...))`가 NULL이 되어 행이 통째로 빠지므로 방어한다:

```sql
(company IS NULL OR NOT (company = ANY($n::text[])))
```

빈 배열이면 해당 절을 아예 붙이지 않는다(필터 미적용).

**적용 지점:** `GET /api/jobs` 라우터가 `get_settings(conn)`으로 두 값을 읽어 `build_list_query`에 넘긴다. 프론트가 아니라 백엔드에 두는 이유 — "전역"이므로 어느 화면에서 부르든 일관되고, Explorer가 전체 공고를 페이지네이션으로 끌어오는 구조라 페이로드도 줄어든다.

**지역 정보가 없는 공고:** 현재 0건. 허용목록이 설정되면 ILIKE가 모두 불일치해 숨겨진다. 특별 분기를 넣지 않는다(수집 오류로만 생길 수 있는 케이스).

## 조회 API — `GET /api/facets`

필터 UI가 쓸 목록. **전역 필터를 적용하지 않는다** — 숨긴 기업도 보여야 다시 켤 수 있다. 이 escape hatch가 없으면 한 번 숨긴 기업을 영영 되살릴 수 없다.

응답:

```json
{
  "regions":   [{"name": "서울", "count": 362}, ...],
  "companies": [{"name": "미스릴", "count": 3}, ...]
}
```

- 지역: `locations`를 콤마로 쪼개 첫 토큰(시/도)을 취하고 **공고 단위로 중복 제거**해 집계. 한 공고가 같은 시/도를 두 번 가져도 1로 센다.
- 기업: `company`별 공고 수. 둘 다 count 내림차순, 동수면 이름순.

## 프론트 — `/filters` 페이지 (전역 필터)

기존 Liquid Glass 카드·`page-head`·저장 버튼(dirty 감지) 패턴을 그대로 따른다. 좌측 rail에 항목 추가(탐색·운영에 이어 세 번째).

**지역 섹션 (포함)**
- 9개 지역 체크박스 + 각 공고 수
- 전부 해제 = 전체 표시(안내 문구로 명시)

**기업 섹션 (제외)** — 255개라 다음이 필수:
- **검색**: 이름 부분일치로 목록 좁히기
- **정렬**: 공고 수 내림차순(영향 큰 기업부터)
- **상태 요약**: "255개 중 3개 숨김"
- **숨긴 기업 먼저 보기** 토글: 되살리기 쉽도록 숨긴 항목을 상단에 모아 표시
- 각 행: 체크 = 표시, **해제 = 숨김**(기본 전체 체크)

**저장**: 하단 저장 버튼. 기존 `GET/PUT /api/settings`를 재사용해 두 필드를 갱신한다(전용 엔드포인트를 새로 만들지 않는다).

## 테스트

**백엔드**
- `build_list_query`: 지역 허용목록 → ILIKE OR 절 생성·파라미터 위치; 기업 숨김목록 → `company IS NULL OR NOT (... = ANY(...))`; **빈 배열이면 절이 붙지 않음**; 기존 필터와 조합 시 `$N` 넘버링 정합.
- `/api/jobs`: 설정의 두 값이 `build_list_query`로 전달되는지.
- facets 쿼리 빌더 + `GET /api/facets` 응답 형태(정렬 포함).
- `Settings` 모델·UPSERT에 새 필드 왕복.

**프론트**
- 지역 체크/해제 → 저장 페이로드 반영, 전부 해제 시 빈 배열.
- 기업 검색으로 목록 좁혀짐, 해제하면 `hidden_companies`에 추가.
- "숨긴 기업 먼저 보기" 정렬 동작, 숨김 개수 요약 문구.
- dirty일 때만 저장 활성(기존 패턴과 동일).

## YAGNI (범위 밖)

- 구/군 단위 전역 필터(시/도만) — Explorer의 2단계 필터가 이미 세션 단위로 담당.
- 수집 시점 차단(비용 절감형) — 되돌리기가 어려워 이번 범위에서 제외.
- 전용 필터 엔드포인트·부분 PATCH — 기존 `PUT /api/settings` 재사용.
- 기업 일괄 선택/해제, 정규식·와일드카드 매칭.
- 설정 동시 편집 충돌 처리(단일 사용자 전제).
